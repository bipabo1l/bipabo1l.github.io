scrapy-cluster分布式url爬虫集群1-爬取多种标签link

2018年1月19日

主题

上一篇文章中，我们已经可以通过分布式集群高效的爬取网站url了，也做了相似性去重处理，然而我们通过测试我们爬得网站并不全面，只爬取了a标签，本文简单介绍如何爬取form表单、iframe标签等url。

测试网站

为了测试爬虫质量，我写了个测试网站，包含了一些标签和链接，源码如下。

<!DOCTYPE html>
<html>

<head>
    <meta charset="UTF-8">
    <title>
        Scrapy Test
    </title>
<script>
function create_url(){
    $.ajax({
        async: true,
        url: "/url_5",
        type: 'get',
        datatype: 'json'
    });
}

$.ajax({
    async: true,
    url: "/url_6",
    type: 'get',
    datatype: 'json'
});
</script>
</head>

<body>

<a href="/url_1" target="_blank">xxx</a>
<form action="/url_2" method="get">
<p>First name: <input type="text" name="fname" /></p>
  <p>Last name: <input type="text" name="lname" /></p>
  <input type="submit" value="Submit" />
</form>
<iframe src="/url_3"><iframe>
<a href="/url_4" style="display:none">xx</a>
<img src="1.png" onclick="create_url()">

</body>
</html>

其中包含7个链接，从url_0到url_6,其中url_5和url_6为ajax中的链接。

源码分析

scrapy-cluster 负责源爬虫的脚本为/crawler/crawling/spiders/目录下的link_spider.py，对源码进行分析。

在link_spider.py 54行，parse()函数中定义了一个LinkExtractor对象，里面给定了allow_domains，allow，deny，deny_extensions四个属性值，我们跟进LinkExtractor类：

发现tags=(‘a’, ‘area’), attrs=(‘href’)，我们在定义LinkExtractor时在tags中加入iframe，form，在attrs加入action，src。
重新运行

scrapy runspider crawling/spiders/link_spider.py

并执行我们的脚本dataTransferAndProcess.py.

已经抓取到其他标签的链接。
